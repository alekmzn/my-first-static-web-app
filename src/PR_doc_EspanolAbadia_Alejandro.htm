Tarea 2.6

 

El CDN que he usado durante todo el curso ha sido Azure y este ofrece también varias opciones de como usar el video en vivo. Una de ellas sería: “Creación de un video en vivo  de Azure Media Services con OBS”

OBS es un software de código abierto para la grabación y transmisión  en internet mantenida por OBS  Project. Actualmente es uno de los software más populares en cuanto streaming se refiere hasta el punto que está actualmente traducido a 41 idiomas.

Para poder utilizarlo con Azure primero de todo hay que tener una cuenta de Azure que podría ser una gratuita para comenzar el proceso.

Deberemos logarnos en Azure portal y por otro lado descargarnos OBS para nuestro sistema operativo local.

 

En Media Services (Azure) debemos seleccionar “Streaming Endpoints” y ponerlo en Running.

Después deberemos ir a “Live Streaming” ir seleccionar “Add live Event” para crear  un nuevo evento

Le podremos nombre, descripción y demás datos propios.

Seleccionaremos Pass-through – no cloud encoding

Seleccionamos la opción RTMP

Deberemos asegurarnos de que este marcado  NO en start live evento para evitar que se empiece a facturar antes de lo tengamos preparado.

Luego vamos a Review + Create  , luego  Create y vovleremos a la lista de eventos en directo.

Al volver al evento en directo, estará detenido,  mantendremos así esta ventana porque la usaremos posteriormente.

Vovlemos a OBS,

Incorporar un origen de video (OBS)

 

Vamos a Sources, Add, seleccionamos Video Captura Device  se abrirá  Create/Select Source

Le damos Add exiting y ok, en el despleable elegimos la entrada de video (como podría ser nuestra cámara o fuente de salida)

 

Incorporar un origen de audio (OBS)

De forma similar al video:   Sources, Add, Audio Input Capture  y se abrirá Create/Select Source. Le damos a Add Exiting y podremos elegir la fuente de audio (micrófono, sonido del pc.. etc)

 

 

Después de esto pasaremos nuevamente a Azure Media Service en el Azure Portal y le damos  a Start.

En RTPM elegimos RTPMS y posteriormente copiamos la URL que aparecerá en el cuadro Input URL.

Volvemos a OBS y tenemos que ir Settings, Stream, Service, Custom  y agregar en Server la dirección URL de RTPMS que hemos copiado ene l paso anterior.

Debemos poner algún valor en Stream Key, no es importante el que pero debe tener un valor.

Vamos a Settings Output, Outpu Mode, Advanced.  Y ahí seleccionamos Straming y ahí elegimos el codificador.  Aquí podríamos elegir si tenemos hardware que lo admita aceleración de GPU por NVENC H.264 o Intel QuickSync H.264, si no lo tenemos  podemos codicar por software con X264. Por simplificar usaremos esta opción.

Codificador X264

Seleciomos Rescale Output y seleccionamos la escala (evento premium 1920*1080 y si es estándar 1280*720, si es de transito básico se puede elegir cualquier resolución)

Seleccionamos la velocidad en bits dependiendo de la resolucion elegida y de la capacidad de la CPU y podemos ir probando hasta conseguir la calidad deseada. Se recomienda después  poner el el campo Keyframe Interval a 2 (2 segundos)

El resto se puede dejar  como viene por defecto.

Ya estamos acabado paramos estableces la configuración de audio:

Pestaña Salida à Configuracionà Audioà Audio Britrate à Ponemos 128 kbps

Configuracionà Audioà Sample Rate à Ponemos 44.1khz

Podemos ir ya al Panel de control y darle a Start Streaming, vamos a la ventana de enevots en directo de Azure Media Services y le damos al vínculos Reload Player y veremos el reproductor en vista previa
